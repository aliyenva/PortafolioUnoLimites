Límites trigonométricos 🌀

Involucran seno, coseno o tangente. El resultado clave es lím (x→0) sen x / x = 1, junto con lím (x→0) (1 − cos x)/x = 0 (Stewart, 2018). Ante una forma 0/0 trigonométrica, se manipula la expresión para que aparezcan estos límites conocidos. En software se relacionan con la aproximación de movimientos suaves en animaciones, donde sen x ≈ x para ángulos pequeños.

  • lím (x→0) sen 3x / x = 3
  • lím (x→0) tan x / x = 1
  • lím (x→0) sen x / 5x = 1/5

Límites exponenciales 🚀

Involucran aˣ o eˣ; los notables son lím (x→0) (eˣ − 1)/x = 1 y lím (x→∞) (1 + 1/x)ˣ = e (Larson y Edwards, 2017). Modelan crecimiento y decaimiento, por ejemplo el decaimiento de una tasa de aprendizaje en un modelo.

  • lím (x→0) (eˣ − 1)/x = 1
  • lím (x→∞) e⁻ˣ = 0
  • lím (x→∞) (1 + 1/x)ˣ = e ≈ 2.71828

Límites con logaritmos 🌳

Involucran ln(x): ln(x) → −∞ cuando x → 0⁺ y ln(x) → +∞ cuando x → ∞, pero crece muy lentamente. El crecimiento logarítmico describe la eficiencia de la búsqueda binaria, O(log n).

  • lím (x→1) ln(x) = 0
  • lím (x→0⁺) ln(x) = −∞
  • lím (x→0) ln(1 + x)/x = 1

Límite trigonométrico

lím (x→0) sen(x)/x = 1

Clave en gráficos por computadora y simulaciones: la función sinc se usa en procesamiento de señales, filtros de audio e interpolación suave.

🎀

Límites exponenciales

lím (x→0) (eˣ − 1)/x = 1 · lím (x→∞) (1 + 1/x)ˣ = e

Fundamentan el interés compuesto continuo y el machine learning: la constante e aparece en la función sigmoide de las redes neuronales.

🍀

Límite logarítmico

lím (x→0) ln(1 + x)/x = 1

La búsqueda binaria divide el espacio a la mitad en cada paso: su eficiencia O(log n) es consecuencia del crecimiento logarítmico.

Definición ε-δ

0 < |x − a| < δ ⟹ |f(x) − L| < ε

Usada en precisión numérica: en algoritmos iterativos (Newton-Raphson, gradiente descendente), ε define el criterio de convergencia.

✏️ Actividad 2 · Ejercicios prácticos
＋Ejercicio 1 · Definición formal ε-δ (5 demostraciones)
a) Demostrar lím (x→4) √x = 2

|√x − 2| = |x − 4| / (√x + 2) ≤ |x − 4| / 2 (porque √x + 2 ≥ 2).

Se elige δ = 2ε ✓
b) Demostrar lím (x→4) (3x − 7) = 5

|(3x − 7) − 5| = 3|x − 4| < ε ⟹ |x − 4| < ε/3.

Se elige δ = ε/3 ✓
c) Demostrar lím (x→−2) (1 − 4x) = 9

|(1 − 4x) − 9| = 4|x + 2| < ε ⟹ |x + 2| < ε/4.

Se elige δ = ε/4 ✓
d) lím (x→3) x² = 9 con δ ≤ 1: cota de |x + 3|

|x² − 9| = |x − 3| · |x + 3|. Con |x − 3| < 1: 2 < x < 4 ⟹ 5 < x + 3 < 7.

Cota superior: 7 · luego δ = mín(1, ε/7)
e) lím (x→−1) (5 − 3x) = 8: estructura tras simplificar

|(5 − 3x) − 8| = 3|x + 1| < ε.

Se llega a |x + 1| < ε/3 (de ahí δ = ε/3)
＋Ejercicio 2 · Límites infinitos (5 límites)
1) lím (x→−3⁺) (x + 2)/(x + 3)

Numerador → −1; denominador → 0⁺.

= −∞
2) lím (x→3⁺) Ln(x² − 9)

x² − 9 → 0⁺ y Ln(0⁺) → −∞.

= −∞
3) lím (x→−2π⁻) x · csc(x)

sen(x) → 0⁻ ⟹ csc(x) → −∞; el factor x es negativo: (negativo)·(−∞).

= +∞
4) lím (x→−3⁺) eˣ/(x − 5)³

El denominador no se anula en x = −3: eˣ → e⁻³ y (x − 5)³ → −512. Límite finito.

= −e⁻³/512 ≈ −0.0000973
5) lím (x→π⁻) cot(x)

cos(π) = −1 y sen(x) → 0⁺.

= −∞

＋Ejercicio 4 · Problema programado 🐍 (concentración C(t))

Para C(t) = (15t + 10)/(3t + 5) y t → ∞ se comparan los grados; el cociente de coeficientes principales es 15/3.

lím (t→∞) C(t) = 5

Forma equivalente: C(t) = 5 − 15/(3t + 5), que muestra que C(t) siempre es menor que 5 y se acerca sin alcanzarlo.

Figura 2 · Convergencia de C(t) hacia su límite: la curva se acerca a la recta y = 5 sin tocarla

🧪 Haz crecer t como en el script 🐍

El deslizador duplica t igual que el bucle del programa: mira cómo la diferencia con el límite se hace chiquitita pero nunca llega a cero.

t1
C(t)3.125
|5 − C(t)|1.875

Script en Python para verificarlo numéricamente:

# Limite de la concentracion C(t) cuando t tiende a infinito
def C(t):
    return (15*t + 10) / (3*t + 5)

epsilon = 0.0001
t = 1.0            # valor inicial
anterior = C(t)

while True:
    t = t * 2      # t crece exponencialmente
    actual = C(t)
    if abs(actual - anterior) < epsilon:
        break
    anterior = actual

print('t final:', t)
print('C(t) aproximado:', actual)
print('Limite analitico:', 5)
print('Diferencia con el limite:', abs(5 - actual))

¿Por qué el programa no llega exactamente a 5? Porque C(t) = 5 − 15/(3t + 5): ese término solo vale cero cuando t es infinito, y el programa usa valores finitos de t. Además se detiene cuando la diferencia entre dos pasos consecutivos es menor que épsilon, no cuando alcanza 5, así que se queda un poco por debajo.

Precisión de punto flotante y límite teórico. La computadora representa los reales con bits limitados, por lo que muchos valores se guardan de forma aproximada. El límite teórico (5) es un valor ideal alcanzado solo en el infinito; el cálculo numérico converge a un valor muy cercano, limitado por el criterio de paro y la representación finita de punto flotante.
