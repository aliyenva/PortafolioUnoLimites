La regla de L’Hôpital 🏥

Sirve para resolver límites con formas indeterminadas 0/0 o ∞/∞: si el límite de f/g da una de esas formas, entonces lím f(x)/g(x) = lím f′(x)/g′(x), siempre que ese segundo límite exista (Stewart, 2018). Es decir, en vez de pelear con el álgebra, comparo las velocidades con que numerador y denominador se acercan a cero o al infinito. Es importante porque resuelve en una línea límites que por factorización serían larguísimos o imposibles, y porque formaliza comparaciones de crecimiento que uso en análisis de algoritmos (¿ln n crece más lento que n? ¡L’Hôpital lo demuestra!).

Tres ejemplos:

  • lím (x→0) sen x/x → 0/0 → lím cos x/1 = 1 (¡el límite notable de la semana 3, ahora en una línea!)
  • lím (x→∞) ln x/x → ∞/∞ → lím (1/x)/1 = 0 → el logaritmo pierde contra la lineal: por eso O(log n) es mejor que O(n) 💻
  • lím (x→0) (eˣ − 1)/x → 0/0 → lím eˣ/1 = 1

⚠️ Solo se aplica si hay indeterminación 0/0 o ∞/∞; si no, se evalúa directo. Y a veces hay que aplicarla dos veces seguidas.

Problemas de optimización 🎯

Buscan el valor máximo o mínimo de una cantidad sujeta a una restricción: minimizar material, costo o tiempo, o maximizar área o ganancia. Son importantes porque casi toda decisión de ingeniería es una optimización disfrazada. Mi paso a paso, con un ejemplo de software:

  • 1. Modelar: quiero minimizar el costo por hora de un servicio en la nube. Con x réplicas del servidor, el costo es C(x) = 2x (cómputo) + 128/x (penalización por latencia/reintentos cuando hay pocas réplicas).
  • 2. Derivar: C′(x) = 2 − 128/x².
  • 3. Puntos críticos: 2 − 128/x² = 0 → x² = 64 → x = 8.
  • 4. Clasificar: C′′(x) = 256/x³ > 0 → es un mínimo. ✓
  • 5. Responder: con 8 réplicas el costo por hora es mínimo: C(8) = 16 + 16 = 32.

Criterios de la primera y la segunda derivada 📈

Primera derivada: donde f′ > 0 la función sube; donde f′ < 0 baja. Si en un punto crítico f′ cambia de + a −, hay máximo local; de − a +, mínimo local. Segunda derivada: f′′ > 0 → cóncava hacia arriba (☕ sonrisa); f′′ < 0 → cóncava hacia abajo; donde f′′ cambia de signo hay punto de inflexión. También clasifica críticos: si f′(c) = 0 y f′′(c) > 0, mínimo; si f′′(c) < 0, máximo (Larson y Edwards, 2017). Juntos permiten graficar una función sin tabular puntos a ciegas: sé dónde sube, dónde se curva y dónde están sus picos.

Ejemplo completo: f(x) = x³ − 3x. f′ = 3x² − 3 = 3(x−1)(x+1) → críticos x = ±1. f′′ = 6x → f′′(−1) = −6 < 0: máximo en (−1, 2); f′′(1) = 6 > 0: mínimo en (1, −2); inflexión en (0, 0). La gráfica queda así:

✏️ Actividad 2 · Ejercicios prácticos

A · Gráficas con los principios de la 1.ª y 2.ª derivada

A.1 · f(x) = 8x⁵ − 5x⁴ − 20x³

Paso 1 · f′: f′(x) = 40x⁴ − 20x³ − 60x² = 20x²(2x² − x − 3) = 20x²(2x − 3)(x + 1)

Paso 2 · Críticos: x = −1, x = 0, x = 3/2.

Paso 3 · Signos de f′: (+) antes de −1 · (−) entre −1 y 0 · (−) entre 0 y 3/2 · (+) después de 3/2.

Paso 4 · Clasificación: en x = −1, f′ cambia + → −: máximo local (−1, 7). En x = 3/2, cambia − → +: mínimo local (1.5, −32.06). En x = 0 no hay cambio de signo: solo tangente horizontal.

Paso 5 · f′′: f′′(x) = 160x³ − 60x² − 120x = 20x(8x² − 3x − 6). Cambia de signo en x = 0 y en x = (3 ± √201)/16 ≈ −0.70 y 1.07: tres puntos de inflexión.

A.2 · f(x) = 5x⁶ − 24x⁴ + 30x⁴

✏️ Primero combino los términos semejantes tal como aparece la función: −24x⁴ + 30x⁴ = 6x⁴, así que f(x) = 5x⁶ + 6x⁴.

Paso 1 · f′: f′(x) = 30x⁵ + 24x³ = 6x³(5x² + 4). Como 5x² + 4 > 0 siempre, el signo de f′ es el signo de x³.

Paso 2 · Monotonía: f′ < 0 si x < 0 (baja) y f′ > 0 si x > 0 (sube) → mínimo absoluto en (0, 0).

Paso 3 · f′′: f′′(x) = 150x⁴ + 72x² ≥ 0 para todo x → cóncava hacia arriba en todo ℝ, sin puntos de inflexión (f′′(0) = 0 pero no cambia de signo).

A.3 · f(x) = x²/(x − 1)

Paso 1 · Dominio y asíntotas: x ≠ 1. Asíntota vertical x = 1; al dividir, f(x) = x + 1 + 1/(x − 1) → asíntota oblicua y = x + 1.

Paso 2 · f′: por cociente, f′(x) = (x² − 2x)/(x − 1)² = x(x − 2)/(x − 1)². Críticos: x = 0 y x = 2.

Paso 3 · Signos: (+) x < 0 · (−) 0 < x < 1 · (−) 1 < x < 2 · (+) x > 2 → máximo local (0, 0) y mínimo local (2, 4).

Paso 4 · f′′: usando f = x + 1 + (x−1)⁻¹: f′′(x) = 2/(x − 1)³ → cóncava ↓ si x < 1, cóncava ↑ si x > 1. No hay inflexión (x = 1 no está en el dominio).

A.4 · f(x) = 3/(x + 2)²

Paso 1 · Dominio y asíntotas: x ≠ −2. Vertical x = −2; horizontal y = 0 (el denominador crece sin cota).

Paso 2 · f′: f = 3(x + 2)⁻² → f′(x) = −6/(x + 2)³. Nunca es cero: no hay extremos.

Paso 3 · Monotonía: si x < −2, (x+2)³ < 0 y f′ > 0 (sube); si x > −2, f′ < 0 (baja). La función "sube hacia" la asíntota y luego baja.

Paso 4 · f′′: f′′(x) = 18/(x + 2)⁴ > 0 siempre → cóncava hacia arriba en todo el dominio, sin inflexiones.

A.5 · f(x) = 1/(x² + x − 6)

Paso 1 · Dominio y asíntotas: x² + x − 6 = (x + 3)(x − 2) → verticales x = −3 y x = 2; horizontal y = 0.

Paso 2 · f′: f′(x) = −(2x + 1)/(x² + x − 6)². Crítico: x = −1/2.

Paso 3 · Signos: el denominador al cuadrado es positivo, así que f′ > 0 si x < −1/2 y f′ < 0 si x > −1/2 → máximo local en (−0.5, −0.16) (¡un máximo con valor negativo, sobre la rama del medio!).

Paso 4 · Concavidad: en las ramas externas (x < −3 y x > 2) la función es positiva y cóncava ↑; en la rama central es negativa y cóncava ↓.

B · Problemas de optimización

B.1 · La hoja de papel con márgenes 📄

Paso 1 · Modelar: sea x el ancho y y el alto del área impresa, con xy = 24 → y = 24/x. La hoja completa mide (x + 2) de ancho (1 cm por lado) y (y + 3) de alto (1.5 cm arriba y abajo).

Paso 2 · Función a minimizar: A(x) = (x + 2)(24/x + 3) = 24 + 3x + 48/x + 6 = 30 + 3x + 48/x

Paso 3 · Derivar e igualar a cero: A′(x) = 3 − 48/x² = 0 → x² = 16 → x = 4 (solo el positivo tiene sentido).

Paso 4 · Clasificar: A′′(x) = 96/x³ > 0 → mínimo. ✓ Entonces y = 24/4 = 6.

Paso 5 · Responder: hoja de ancho 4 + 2 = 6 cm y alto 6 + 3 = 9 cm.

Hoja de 6 cm × 9 cm (área mínima: 54 cm²)
B.2 · Dos números positivos con producto 288 🔢

Paso 1 · Modelar: xy = 288 y quiero minimizar S = 2x + y. Despejo y = 288/x.

Paso 2 · Función: S(x) = 2x + 288/x

Paso 3 · Derivar: S′(x) = 2 − 288/x² = 0 → x² = 144 → x = 12.

Paso 4 · Clasificar: S′′(x) = 576/x³ > 0 → mínimo. ✓ Entonces y = 288/12 = 24.

Los números son 12 y 24 (suma mínima: 2·12 + 24 = 48)
B.3 · El cuarto de base cuadrada más barato 🏠

Paso 1 · Modelar: base cuadrada de lado x y altura h, con volumen x²h = 1000 → h = 1000/x².

Paso 2 · Costo: techo + piso = 2 caras de x² a ₡2000: 4000x². Paredes = 4 caras de xh a ₡1800: 7200xh.

C(x) = 4000x² + 7200x·(1000/x²) = 4000x² + 7 200 000/x

Paso 3 · Derivar: C′(x) = 8000x − 7 200 000/x² = 0 → x³ = 900 → x = ∛900 ≈ 9.65 m.

Paso 4 · Clasificar: C′′(x) = 8000 + 14 400 000/x³ > 0 → mínimo. ✓

Paso 5 · Responder: h = 1000/(∛900)² ≈ 10.73 m.

Base de ∛900 ≈ 9.65 m por lado y altura ≈ 10.73 m (costo ≈ ₡1 118 689)

💭 ¿Qué aprendí?

A resolver indeterminaciones con L’Hôpital, a plantear y resolver problemas de optimización con restricciones, y a graficar funciones usando los criterios de la primera y segunda derivada.

🌱 ¿Cómo lo aprendí?

Siguiendo un método fijo para cada gráfica (dominio → f′ → f′′ → asíntotas) y verificando mis trazos con las gráficas que construí en la página, además de la práctica guiada con la IA.

🛠️ ¿Cómo lo aplico?

En comparaciones de crecimiento de algoritmos (ln n vs n vía L’Hôpital) y en decisiones de recursos, como encontrar el número de réplicas que minimiza el costo de un servicio.

Para cerrar el módulo de derivadas viendo que no son solo reglas: son una herramienta para decidir, predecir y dibujar el comportamiento completo de una función.
